# Patterns

> Plantilla. Se vuelve canonica solo dentro de un proyecto activo.

## Proposito

Registrar patrones aprobados y anti-patrones que guian implementaciones futuras.

## Como llenarlo

Escribir patrones concretos, verificables y utiles para agentes. Evitar normas genericas que ya vivan en constraints globales.

## Ejemplo minimo

```text
Patron: Validaciones de negocio en servicios de dominio.
Anti-patron: Validar permisos solo en componentes UI.
```
