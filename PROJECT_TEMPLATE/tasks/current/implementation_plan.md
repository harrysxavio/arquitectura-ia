# Implementation Plan

> Plantilla. Se vuelve canonica solo dentro de un proyecto activo.

## Proposito

Definir pasos de ejecucion para tareas que requieren coordinacion, validacion o control de riesgo.

## Como llenarlo

Usar cuando la tarea sea multiarchivo, estructural, ambigua o tenga riesgo moderado/alto. Omitir para cambios triviales.

## Ejemplo minimo

```text
1. Actualizar spec.
2. Implementar validacion en servicio.
3. Agregar tests.
4. Registrar decision si cambia el patron de validacion.
```
